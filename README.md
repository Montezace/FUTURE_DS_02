# FUTURE_DS_02: Customer Retention & Churn Analysis

## 🔍 Project Overview
This repository contains my submission for **Task 2** of the Data Science & Analytics Track at Future Interns. The objective of this project is to analyze customer subscription data to identify churn patterns, uncover key retention drivers, and provide actionable business recommendations to reduce customer drop-off.

## 🎯 Business Value
Customer retention is core to the survival of SaaS and subscription-based businesses. By identifying *who* is leaving and *why*, this analysis helps business stakeholders redirect product and marketing efforts to save Monthly Recurring Revenue (MRR) and increase Customer Lifetime Value (LTV).

## 🛠️ Tools & Technologies Used
- **Python:** Primary programming language for data manipulation.
- **Pandas:** Data cleaning, boolean mapping, and cohort grouping.
- **Matplotlib:** Data visualization for retention health and segment drop-offs.
- **Google Colab (Jupyter Notebook):** Interactive execution environment.
- **ipywidgets:** For building a dynamic, dataset-agnostic analysis pipeline.

## 📂 Repository Structure
- `FUTURE_DS_02_Notebook.ipynb`: The Google Colab notebook containing the automated data processing and KPI generation script.
- `FUTURE_DS_02_Report.html`: The client-ready analytical report generated by the script, featuring actionable retention strategies.
- `data.csv`: The anonymized customer churn dataset used for this analysis.

## 🚀 Key Insights Generated
- **Overall Platform Health:** Computed the baseline churn rate to establish a metric for future retention campaigns.
- **High-Risk Segments:** Identified the specific customer segment experiencing the highest drop-off rate, providing product teams with a clear target for intervention workflows.
